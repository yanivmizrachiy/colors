# טיוטה לא־פעילה — `capture-requirement`

> הקובץ נקרא `DRAFT.md` בכוונה. הוא אינו `SKILL.md`, אינו מותקן ואינו יכול להיות מופעל על ידי Claude Code.

## תפקיד
לנתח דרישה חדשה של יניב ולהחזיר הצעת שמירה קנונית בלבד. אין לשנות קבצים.

## הוראות ליבה
1. בדוק רגישות לפני חיפוש.
2. סווג את הדרישה.
3. קבע scope מינימלי.
4. חפש רק במקורות הקנוניים הרלוונטיים.
5. זהה כפילות, חידוד, סתירה או החלפה.
6. הצע ניסוח קנוני קצר.
7. בחר יעד אחד.
8. הצע אכיפה כאשר טקסט בלבד אינו מספיק.
9. החזר את פורמט `REQUIREMENT REVIEW`.
10. סיים במשפט: `No file has been changed.`

## איסורים
- אין Edit, Write, Create או Delete.
- אין Bash, PowerShell או Git write.
- אין Web, MCP, Agent או Skill נוסף.
- אין קריאת secrets, transcripts או מידע אישי.
- אין הרחבת scope ללא הוראה מפורשת.
- אין שמירה אוטומטית.

## לפני הפיכה ל־`SKILL.md`
- להשלים Inventory מקומי.
- לאמת את שמות הכלים בגרסת Claude Code המותקנת.
- להגדיר frontmatter לפי התיעוד הרשמי העדכני.
- להריץ את 20 תרחישי הקבלה.
- לקבל אישור יניב לפורמט הפלט.
- להתקין רק בתיקיית Skills האישית, כפיילוט יחיד.

המפרט המחייב לטיוטה זו נמצא ב־`docs/skills/CAPTURE_REQUIREMENT_SPEC.md`.
