# אינדקס מחקר הריפוים

## מטרת האינדקס
לרכז במקום אחד את מצב המחקר, העדיפות, איכות הראיות והצעד הבא לכל ריפו. זהו לוח בקרה מחקרי, לא רשימת משימות יישום.

## סטטוסים
- **לא התחיל** — טרם נאספו ראיות.
- **סריקה ראשונית** — נקראו קובצי כניסה ונבדקה פעילות.
- **מחקר פעיל** — PRs, commits, בדיקות וזרימות עבודה נבחנים.
- **טיוטת דוח** — קיימות מסקנות מבוססות אך חסר אימות מקומי או מדידה נוספת.
- **דוח מבוסס** — יש ראיות מכמה מקורות ושאלות פתוחות מוגדרות.
- **ממתין למידע מקומי** — GitHub אינו מספיק להשלמת המסקנות.

## השוואה ראשונית בין שני הדוחות הראשונים
- `mathmath`: context פתיחה רזה, אך בדיקות סביבת פריסה ו-E2E פחות אכיפתיות.
- `parabula-next`: context פתיחה כבד יותר, אך contracts ו-CI אכיפתיים ורחבים מאוד.
- השאלה המחקרית המרכזית: היכן נמצא האיזון הנכון בין זיכרון קבוע, gates אוטומטיים, זמן CI ומהירות ביצוע.

## מידע שייאסף לכל ריפו
- מטא־דאטה: גודל, פעילות, visibility וענף ברירת מחדל.
- מקור אמת ומודל זיכרון.
- scripts ופקודות בדיקה.
- CI, preview ופריסה.
- PRs ותיקוני המשך.
- שגיאות פרודקשן ורגרסיות.
- מעורבות ידנית של יניב.
- context/session ונתוני שימוש מקומיים.
- מועמדים עתידיים לשיפור, ללא יישום.

## כללי עדכון
1. אין לסמן דוח כמבוסס לפני שיש יותר ממקור אחד.
2. כל שינוי סטטוס חייב לכלול צעד הבא.
3. כאשר חסר מידע מקומי, יש לציין בדיוק איזה מידע נדרש.
4. אין להוסיף ריפו לרשימת היישום; שלב זה מחקר בלבד.
5. מסקנה חוצת־פרויקטים דורשת לפחות שני דוחות עומק.
