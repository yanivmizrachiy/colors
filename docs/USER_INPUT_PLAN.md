# תוכנית קבלת מידע מיניב ומהמחשב

## מטרה
לאסוף רק את המידע שנדרש לבניית תוכנית עבודה אישית עם Claude Code, במינימום סיכון ובמקסימום ערך מחקרי.

## עיקרון
לא מעלים את כל `%USERPROFILE%\.claude` ולא משתפים raw transcripts. האיסוף יהיה מדורג, מסונן וניתן לבדיקה לפני שיתוף.

## שכבה 1 — תמונת עבודה אישית
נבקש מיניב לתאר או להדגים:
1. אילו פרויקטים הוא פותח הכי הרבה ב-Claude Code.
2. אילו משימות חוזרות: פיצ'רים, תיקונים, עיצוב, מחקר, פריסה, Git, קבצים או אינטגרציות.
3. מה הוא רוצה שהמערכת תעשה אוטומטית ומה הוא רוצה לאשר בעצמו.
4. אילו טעויות או תסכולים חוזרים.
5. מה נחשב תוצאה טובה: מהירות, איכות, עצמאות, UI, בדיקות, תיעוד או פריסה.
6. באילו מחשבים וכלים הוא עובד.
7. אילו אינטגרציות פעילות: GitHub, Vercel, DB, MCP, דפדפן, IDE ועוד.

## שכבה 2 — Inventory מקומי מסונן
קובץ ביקורת עתידי יאסוף רק:
- גרסת Claude Code ומיקום ההתקנה.
- Windows/PowerShell/Node/Git versions.
- שמות MCP ו-Plugins, ללא tokens או config סודי.
- רשימת Skills, Hooks, Agents ו-commands לפי שם ונתיב.
- רשימת פרויקטים לפי path, פעילות ו-Git remote מסונן.
- אינדקס sessions: מזהה, פרויקט, תאריך, גודל ומספר שורות בלבד.
- סטטיסטיקות usage זמינות.
- inventory של `CLAUDE.md`, rules ו-settings לפי path/size/hash.

## שכבה 3 — דפוסי פרומפטים מסוננים
רק לאחר סקירת שכבה 2:
- נושאי בקשות חוזרים.
- הוראות שחוזרות.
- תיקונים שחוזרים.
- מילות מפתח של build/test/deploy/Git.
- ללא קוד מקור, סודות או תוכן מלא של שיחות.

## שכבה 4 — שיחות נבחרות
רק שיחות שנבחרו במפורש:
- משימה מוצלחת במיוחד.
- משימה שהסתבכה.
- משימה עם תיקוני המשך רבים.
- משימה ארוכה עם שימוש כבד בהקשר.

עדיפות ל-`/export` קריא ולא לעיבוד ישיר של JSONL פנימי.

## מידע שאסור לאסוף או להעלות
- `.credentials.json`.
- `.claude.json` גולמי.
- `.env` או secrets.
- API keys, OAuth tokens, cookies וסיסמאות.
- נתוני תלמידים או מידע אישי.
- raw transcripts מלאים כברירת מחדל.
- browser profiles או קישורי עריכה פרטיים.

## שאלות שנענה עליהן בעזרת האיסוף
1. כמה פרויקטים פעילים באמת?
2. כמה context נטען בתחילת עבודה?
3. אילו קבצים חוזרים בכל session?
4. אילו MCP או Plugins מוסיפים עומס ללא שימוש?
5. אילו פעולות חוזרות מתאימות בעתיד ל-Skill או Hook?
6. מתי session נמשך יותר מדי ומתי כדאי clear/compact/handoff?
7. מה ההבדל בין פרויקטים מצליחים לפרויקטים שמסתבכים?
