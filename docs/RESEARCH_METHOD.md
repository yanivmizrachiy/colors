# שיטת המחקר

## גבולות
- כל ריפו אחר נבדק בקריאה בלבד.
- אין שינוי קוד, הגדרות, ענפים, Issues, PRs או פריסות בריפוים הנחקרים.
- אין איסוף secrets, מידע תלמידים, raw logs או מידע אישי.

## סדר חקירת ריפו
1. פעילות אחרונה, גודל וחשיבות.
2. מטרת המוצר והסטאק.
3. `CLAUDE.md`, `AGENTS.md`, Rules וקובצי memory.
4. scripts, commands, preview ו־launch.
5. CI, tests, lint, build ו־deployment.
6. commits ו־PRs שנוצרו עם Claude Code.
7. תקלות חוזרות, rollback ורגרסיות.
8. הוראות שחוזרות ויכולות להפוך לאוטומציה.
9. היכן Claude מצליח והיכן הוא נוטה להסתבך.
10. השפעה משוערת על זמן, איכות והקשר.

## פורמט ממצא
לכל ממצא נשמור: מקור, תאריך, מה נצפה, למה זה חשוב, יתרון, סיכון, מועמדות ל־Memory/Rule/Skill/Hook/Agent/Script/Test, והאם נדרש אימות נוסף.

## תהליך קבלה
מחקר → השערה → החלטה → פיילוט → מדידה → הרחבה.
