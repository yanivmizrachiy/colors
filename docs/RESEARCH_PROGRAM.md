# תוכנית המחקר העמוקה של קלוטקורד

> מסמך מחקר היסטורי ומתודולוגי. הוא אינו דף כללים ואינו גובר על `CLAUDE.md`.

## מטרת המסמך
לתעד כיצד נחקרה העבודה של יניב עם Claude Code, כיצד נשמרו ראיות, כיצד הופרדו עובדות מהסקות, ואיך נבנתה תוכנית אישית בלי לקפוץ ליישום מוקדם.

## מצב
שלב המחקר הראשוני הושלם. קלוטקורד נמצא כעת בשלב ניקוי, מחקר הרחבות עדכני ופיילוט מדיד. המצב החי מופיע רק ב־`CURRENT_STATE.md`.

## שאלות המחקר המרכזיות
1. באילו סוגי משימות יניב משתמש ב־Claude Code בפועל?
2. אילו שלבים חוזרים בכל פרויקט: חקירה, תכנון, קוד, בדיקות, preview, Git, PR ופריסה?
3. היכן מתבזבז זמן על הסברים חוזרים, קריאות קבצים, פלטים גדולים או תיקוני המשך?
4. אילו טעויות חוזרות נובעות מחוסר הקשר, מקור אמת לא ברור, בדיקות חסרות או סיום מוקדם?
5. אילו דפוסים מוצלחים כבר קיימים וראוי לשמר?
6. איזה מידע צריך להיות גלובלי, פרויקטלי, לפי משימה או מחוץ להקשר הקבוע?
7. אילו תהליכים עשויים להתאים ל־Memory, Rule, Skill, Hook, Agent, Plugin, MCP, Script או Test?
8. איך למדוד שיפור אמיתי ולא רק תחושה?

## יחידות המחקר
### שכבה א — GitHub
- מבנה הריפו והסטאק.
- קובצי `CLAUDE.md`, `AGENTS.md`, Rules, memory ותיעוד.
- commits, PRs, תיקוני המשך ופריסות.
- tests, lint, build, preview ו־CI.
- אירועי rollback, רגרסיות ותקלות production.

### שכבה ב — סביבת Claude Code המקומית
תיבדק באמצעות inventory מסונן בלבד:
- גרסה והתקנה.
- הגדרות משתמש ופרויקט.
- MCP, Plugins, Skills, Hooks ו־Agents.
- אינדקס פרויקטים ושיחות ללא תוכן רגיש.
- פקודות וכלים חוזרים ללא transcripts גולמיים.
- `/usage`, `/context`, `/doctor`, `/memory`, `/config`, `/mcp` ו־`/plugin`.

### שכבה ג — דרך העבודה של יניב
- מה יניב מבקש שוב ושוב.
- מה הוא מעדיף שהמערכת תעשה לבד.
- היכן הוא נאלץ להתערב ידנית.
- אילו סוגי תוצאה נחשבים אצלו מוצלחים.
- אילו תקלות או אי־הבנות חוזרות בשיחות.

### שכבה ד — מקורות חיצוניים
- תיעוד רשמי עדכני של Anthropic.
- marketplaces רשמיים וקהילתיים.
- repositories של Skills/Plugins/Frameworks עם קוד, גרסאות והיסטוריה ניתנים לבדיקה.
- מאמרים ומחקרים על עלויות, בטיחות, prompt injection ואיכות תוצר.
- סרטונים ופוסטים משמשים כרעיונות בלבד עד אימות מול מקור ראשוני.

## שלבי העבודה
1. מיפוי הריפוים לפי פעילות, גודל, מורכבות והשפעה.
2. דוח עומק לכל ריפו מרכזי.
3. מטריצת דפוסים חוצי־פרויקטים.
4. מיפוי עבודה מקומית מסונן.
5. ניתוח צרכים והרגלים של יניב.
6. מחקר עדכני של הרחבות ודרכי עבודה.
7. דירוג מועמדים לפי ערך, סיכון, context ותחזוקה.
8. פיילוטים קטנים והפיכים.
9. תוכנית עבודה סופית ומדורגת.

## תוצרים
- `docs/projects/<repo>.md` — דוח עומק לכל ריפו.
- `research/REPOSITORY_INDEX.md` — דירוג ומצב המחקר.
- `research/CROSS_PROJECT_MATRIX.md` — דפוסים חוצי־פרויקטים.
- `research/OPPORTUNITY_REGISTER.md` — הזדמנויות עתידיות.
- `docs/OFFICIAL_CLAUDE_CODE_FINDINGS.md` — מחקר רשמי עדכני.
- `research/EXTENSION_CANDIDATES.md` — מועמדי Skills, Plugins, Hooks ו־MCP.
- `docs/PERSONAL_WORK_SYSTEM_PLAN.md` — התוכנית האישית הסופית.

## תנאי איכות מחקריים
- כל טענה מסומנת כעובדה, הסקה או השערה.
- לכל עובדה חיצונית יש מקור ותאריך.
- לא מסיקים מדוגמה אחת דפוס כללי.
- לא ממליצים על הרחבה לפני בדיקת תדירות, חלופה פשוטה יותר, הרשאות, עלות context, תחזוקה ו־rollback.
- אין מידע רגיש בריפו הציבורי.

## הגדרת השלמה
המחקר ייחשב בשל לתוכנית סופית רק לאחר שיש ראיות מהפרויקטים של יניב, inventory מקומי מסונן, מחקר עדכני של הרחבות, מדדים מפיילוטים וחלופות ברורות לכל המלצה.
