1. הוראות קרובות לקוד שהן משפיעות עליו.
2. אין צורך לטעון חוזי web כאשר עובדים באזור אחר.
3. מתאים למונוריפו או ריפו עם כמה מוצרים עצמאיים.
4. קובץ מקומי קצר יכול לגבור על כללים כלליים בלי להעמיס על השורש.

## סיכונים
1. הוראות root ו-subdirectory עלולות להצטבר ולהיות סותרות, משום ש-Claude מצרף אותן ולא מבטל את הקודמות.
2. קיום גם `CLAUDE.md` וגם `AGENTS.md` עלול ליצור כפילות אם אין import או pointer יחיד.
3. בלי אינדקס root ברור, סוכן עלול להתחיל בתיקייה הלא נכונה או לפספס את ההוראות המקומיות.
4. אין כרגע מספיק ראיות על tests, CI, Git workflow או תדירות שימוש.

- בפרויקט עם כמה אפליקציות: root CLAUDE קצר מאוד + הוראות מקומיות בכל אפליקציה.

- יש להפעיל Claude מתוך תיקיית הפרויקט המתאימה, לא תמיד משורש monorepo.
