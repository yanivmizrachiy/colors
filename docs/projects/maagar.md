## ממצא מרכזי
`maagar` הוא המודל הטוב ביותר שנבדק להעברת ידע מהקשר של המודל לקוד דטרמיניסטי. הוא גם מלמד שהאכיפה צריכה להיות מינימלית ומתוחזקת: validator ישן או דוח stale עלולים להפוך למקור שגיאה בפני עצמם.

## לקחים למערכת האישית של יניב
- דרישה שחוזרת פעמיים ומדידה באופן דטרמיניסטי → test/guard, לא עוד שורה ב-CLAUDE.
- פעולה על קבצים רבים → dry-run + report + apply מפורש.
- אין metadata מומצא; משתמשים ב-unknown או עוצרים.
- audit outputs → artifacts או דוח זמני, לא commit קבוע.
- source-of-truth אחד לכל תחום; pointers דקים בלבד.
- PR description צריך לכלול: scope, אמת/חסימות, בדיקות, מה לא השתנה ו-rollback.
