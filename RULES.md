# RULES.md — פרויקט colors

## מטרת הפרויקט

`colors` הוא ריפו ואתר חיצוני קבוע לספריית עיצוב בעברית RTL עבור אתרים ש־GPT יוצר.

המטרה: לאפשר בחירה מהירה של צבעים, רקעים, כותרות, כפתורים, טבלאות, מסגרות, גופנים, גדלי גופנים ומערכות עיצוב — עם כפתור העתקה ל־Prompt מוכן ל־GPT.

## גבול אחריות חשוב

`colors` הוא ספריית עיצוב ומחסן פרומפטים חכמים. הוא לא אמור לערבב בתוכו את כל האתרים שהמשתמש יבנה בעתיד.

מפעל האתרים / עוזר GPT לבניית אתרים צריך להיות ריפו נפרד או שכבה נפרדת וברורה, כדי שלא יהיה בלבול בין:

- ספריית עיצוב וצבעים.
- אתר חדש שנוצר עבור מטרה מסוימת.
- תשתית אוטומציה שמייצרת אתרים.
- תוכן אמיתי של אתר אחד לעומת אתר אחר.

- כל אתר חדש חייב להיות מותאם היטב לנייד.

- כל אתר חייב להיות מזוהה לפי `siteId` ייחודי.
- כל אתר חייב להיות בתיקייה ייחודית.
- כל אתר חייב לכלול קובץ הגדרות ייחודי.

- כל אתר חייב לשמור קישור ציבורי ייחודי.
- כל פעולה של GPT חייבת לזהות במדויק את אתר היעד לפני שינוי.

- כל אתר וכל עריכה עתידית חייבים לשמור הפרדה ברורה בין HTML, CSS ו־JavaScript.

## מבנה חובה לכל אתר עתידי

ברירת המחדל לכל אתר עתידי:

```text
sites/{siteId}/
  index.html        # מבנה האתר בלבד
  styles.css        # עיצוב האתר בלבד
  script.js         # התנהגות/אינטראקציה בלבד
  site.config.json  # זהות, סטטוס, קישור, מטא־דאטה
  README.md         # תיעוד האתר
```

## כלל הפרדה בין קבצים

- HTML מיועד למבנה.
- CSS מיועד לעיצוב.
- JavaScript מיועד להתנהגות.
- JSON מיועד להגדרות ומטא־דאטה.
- README מיועד לתיעוד.

אין לערבב CSS גדול בתוך HTML. אין לערבב JavaScript גדול בתוך HTML. חריג קטן מותר רק אם יש סיבה טכנית ברורה ומתועדת, אבל ברירת המחדל המחייבת היא קבצים נפרדים.

## מניעת בלבול בין אתרים

לפני כל שינוי באתר עתידי, עוזר GPT חייב לוודא:

- מהו `siteId` המדויק.
- מה שם האתר.
- מה התיקייה של האתר.
- מה הקישור הציבורי של האתר.
- אילו קבצים מותר לשנות.
- אילו קבצים אסור לשנות.

פעולת עדכון תקינה צריכה להתייחס במפורש ליעד, לדוגמה:

```json
{
  "operation": "update_site",
  "siteId": "math-angle-worksheet",
  "targetPath": "sites/math-angle-worksheet/",
  "changedFiles": [
    "sites/math-angle-worksheet/index.html",
    "sites/math-angle-worksheet/styles.css"
  ],
  "publicUrl": "https://yanivmizrachiy.github.io/gpt-sites/sites/math-angle-worksheet/"
}
```

## מה עדיין לא סגור

- יש לנקות סופית ניסוחים ישנים מתוך `design-tokens.json` ו־`site.config.json` אם הם עדיין קיימים.
- יש להריץ שוב את `bash ~/colors_ultra_check.sh` אחרי הניקוי.
- יש לוודא שמתקבל:
  - `VALIDATION_OK`
  - `COLORS_DOCTOR_OK`
  - `ZERO_DEMO_AUDIT_OK`
  - `LIVE_VERIFY_OK`
  - `COLORS_ULTRA_CHECK_DONE`
- יש לבדוק בטלפון אמיתי שהאתר נפתח, שאין גלילה אופקית, והעתקת Prompt עובדת בפועל.
- אין כרגע בריפו הזה יצירה מאומתת של ריפו נפרד בשם `gpt-sites`; אם ייבנה מפעל אתרים מלא, יש לבצע זאת כריפו נפרד או כמבנה נפרד שמוגדר במפורש.
