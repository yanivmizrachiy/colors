# colors — ספריית העיצוב של יניב

`colors` הוא ריפו ציבורי ואתר GitHub Pages לספריית עיצוב בעברית RTL עבור אתרים ש־GPT יוצר.

## קישורים

- ריפו: `https://github.com/yanivmizrachiy/colors`
- אתר חי: `https://yanivmizrachiy.github.io/colors/`
- Smart Builder: `https://yanivmizrachiy.github.io/colors/smart-builder.html`
- בדיקת נייד: `https://yanivmizrachiy.github.io/colors/qa-mobile.html`

## נוצר בפועל לפי מקור האמת של הפרויקט

- ריפו `yanivmizrachiy/colors` קיים.
- GitHub Pages מוגדר לפי `RULES.md`.
- קיימים `index.html`, `styles.css`, `script.js`, `site.config.json`.
- קיימים רכיבי עיצוב אמיתיים עם כפתורי צפייה והעתקה.
- קיימת תצוגה מוגדלת.
- קיימת העתקת Prompt/CSS/צבע/קישור.
- קיימים חיפוש, סינון, מועדפים ונבחר לאחרונה.
- קיימת בדיקת validate לפי `RULES.md`.

## גבול אחריות

הריפו הזה הוא ספריית עיצוב ומחסן פרומפטים חכם.

מפעל האתרים העתידי של GPT, שמייצר עוד ועוד אתרים ציבוריים עם קישורים חיצוניים קבועים, צריך להישמר בנפרד או במבנה מוגדר וברור כדי למנוע ערבוב בין אתרים.

## דרישות שנוספו למפעל האתרים העתידי

- כל אתר חייב לקבל `siteId` ייחודי.
- כל אתר חייב להיות בתיקייה ייחודית.
- כל אתר חייב לקבל קישור ציבורי ייחודי.
- אין לערבב תוכן, עיצוב או היסטוריה בין אתרים.
- כל אתר חייב להיות מותאם לנייד.
- כל אתר חייב להיות גרפי, איכותי, נוח וקריא.
- כל אתר חייב לשמור הפרדה בין:
  - `index.html` — מבנה.
  - `styles.css` — עיצוב.
  - `script.js` — התנהגות.
  - `site.config.json` — הגדרות ומטא־דאטה.
  - `README.md` — תיעוד.
